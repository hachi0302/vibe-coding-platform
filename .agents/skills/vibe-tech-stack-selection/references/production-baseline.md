# 可上线基线

选型不是只选框架。只要项目包含持久化数据、对外 API、身份信息或关键业务，方案必须按项目阶段声明最低上线基线。

| 项目阶段 | 最低基线 |
| --- | --- |
| 原型 | 健康检查、配置与密钥不入库、可重复构建、基础错误日志；可明确标注不建议直接生产上线 |
| 正式业务 | 环境隔离、数据库迁移、结构化日志与关联 ID、认证授权、备份责任、自动化测试、回滚方式 |
| 长期/关键业务 | 指标与告警、分布式追踪、漏洞与依赖治理、限流/熔断策略、容量与故障演练、运行手册、审计与合规要求 |

## 输出要求

每项上线基线都标识：已有平台复用、项目需补建、或本阶段暂缓。不得把某个云厂商、CI、监控产品或安全扫描工具强行指定给所有项目。
