# 磨砂玻璃工作台主题设计

## 目标

为 Vibe Coding Platform 的主工作台新增一个可由用户选择的“磨砂玻璃（预览）”主题。它只改变窗口外壳、顶部栏、侧栏和分栏容器的视觉表现；现有会话、终端、项目工厂和业务行为保持不变。

## 用户体验

- 设置的“主题”下拉菜单新增“磨砂玻璃（预览）”。用户主动选择后才启用，选择任一既有主题立即恢复既有外观。
- 主工作台呈现深色半透明分层：窗口底色可透出桌面背景，顶部栏和侧栏使用较强模糊，主分栏使用较轻模糊与细边框。
- 代码、终端、长会话正文保持深色实底或高不透明度，不牺牲可读性。
- 现有 agent 品牌色、运行状态、分屏与拖拽高亮保留，仅让其在玻璃表面上有更柔和的光晕。

## 技术边界

- 增加 `glass` 主题枚举、localStorage 持久化、设置菜单选项和原生菜单选项。
- Tauri 主窗口允许透明背景；经典主题仍通过现有不透明 CSS 绘制完整背景，因此视觉和交互不变。
- 主题 CSS 使用 `:root.theme-glass` 和 `.theme-glass .app` 精确限定。不得重写会话内容、终端渲染、项目工厂或设置页的布局。
- 不新增 UI 库，不修改当前工作区中已有的项目初始化未提交文件。

## 视觉规则

- 底层：蓝黑半透明背景，配合低对比度的冷色光斑，不使用高饱和渐变或大面积发光。
- 顶部栏与侧栏：`backdrop-filter: blur(...) saturate(...)`，半透明深色底、1px 低透明亮边。
- 主面板：比侧栏更实，确保会话与终端文本可读；多分屏之间用细亮分隔线，不改尺寸和拖拽命中区。
- 悬停、选中与运行状态沿用既有品牌/状态色，光晕只在已有交互状态出现。

## 验收标准

1. 设置中可选择并持久化“磨砂玻璃（预览）”；重启后仍保持选择。
2. 切换回浅色、深色、跟随系统、Codex、Dracula 时，现有主题行为不变。
3. 玻璃主题下主工作台的顶部栏、侧栏、主区和分屏容器可见分层磨砂效果；终端和正文文本仍清晰。
4. `npm run build` 与涉及主题选择的测试通过。
5. 不修改当前工作区中已有的项目初始化未提交文件。
