# 后端自测

## 触发条件

- 已发生后端代码、接口、数据、异步或第三方调用变更，且用户要求或交付需要自测。
- 只使用项目中真实存在的测试基座、命令和安全环境；不把配置线索当成可执行能力。

## 测试选择

| 变化 | 首选测试 |
|---|---|
| 纯计算/规则 | 单元测试，使用项目既有断言与 mock 框架 |
| Service 编排 | 依赖可控的单元测试；必要时集成测试 |
| Controller/API | 请求校验、鉴权、响应与错误契约测试 |
| DB/迁移 | Repository/DAO 集成测试与迁移验证 |
| MQ/异步 | 消息结构、幂等、重试、重复消费和终态测试 |
| 第三方 | 官方契约样例、签名/验签及失败响应测试 |

## 执行步骤（Pipeline）

1. 读取后端规则、详设与现有同类测试，沿用项目测试基座。
2. 先写并运行失败测试，证明测试命中目标行为。
3. 覆盖正常、边界、null/空、异常、原 Bug case 和受影响回归。
4. 运行定向测试，再执行 `{{后端完整验证命令}}`；记录真实命令、退出码和测试数。
5. 测试数据必须隔离、可重复；不得调用生产环境或写入不可恢复数据。
6. 按项目约定保留有长期价值的测试，删除一次性诊断代码。

## 两轨测试

1. 离线行为测试：锁定规则、分支、边界、错误与历史回归；第三方和基础设施按项目已有方式 mock。
2. 集成/真实链路测试：只有项目已有安全环境、数据隔离和接入方式时执行；记录环境、输入、响应和数据结果，绝不碰生产。

离线测试不能伪造 production 逻辑后测试自造实现；真实链路成功也不能替代边界与异常测试。

## 禁止事项

- 禁止删除失败测试、放宽业务断言、扩大 mock 或更新快照来求绿。
- 禁止调用生产环境、写入不可恢复数据、伪造项目不存在的测试框架或命令。
- 禁止将编译成功、mock 调用次数或单一 happy-path 当作完整行为证明。

## 可执行验证

- 仅执行项目资源中确认存在的定向测试、完整验证、迁移或契约检查命令；记录命令、cwd、退出码、通过数和范围。
- 每项变化至少映射正常、边界、异常、原 Bug 和受影响回归中的适用 case；不适用项写入待补信息。

## 失败处理

- production 缺陷：保留失败 case，修复最小代码后重新全量验证。
- 测试错误：说明错误原因并修正测试，禁止放宽业务断言。
- 环境阻断：记录命令和错误，列出仍未验证的业务风险，不写“应该通过”。

## 交付证据

必须报告测试类路径、每类 case、命令结果和测试文件处置。仅口头说明“测过”或“编译通过”不算完成。

## 完成 Gate

- 正常、边界、null/空、异常、原 Bug 和相关回归均有独立 case。
- 定向测试与项目完整验证给出真实退出码/通过数。
- 数据、消息、状态或外部契约变化有对应结果断言。
- 测试文件已按项目约定保留或删除，工作区无临时诊断垃圾。
