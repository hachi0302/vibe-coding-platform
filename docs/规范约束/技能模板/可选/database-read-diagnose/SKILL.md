只记录配置键、文件路径和脱敏连接状态。不得把 URL 中的账号密码、token、证书或生产数据样本写入 skill、日志或文档。

## 执行流程

1. 明确环境、业务号/主键、时间范围、目标表和最少所需字段。
2. 读取物理模型、Repository/DAO 与迁移脚本，确认表字段和关联关系。
3. 验证连接是否为已授权只读身份；只允许 `SELECT`、`WITH ... SELECT`、`SHOW`、`DESCRIBE`、`EXPLAIN`。
4. 先查主记录，再按真实外键或业务键追关联记录；限制字段、条件和行数。
5. 将查询结果与代码分支、状态转换和日志时间线交叉验证。
6. 输出脱敏 SQL、结果摘要、代码证据和结论可信度。

禁止执行 INSERT、UPDATE、DELETE、MERGE、DDL、存储过程、锁表或会改变会话/数据状态的语句。生产环境即使只读也必须遵守项目额外授权规则。

## 完成 Gate

- 查询目标、环境、条件、字段和行数均为解决当前问题所必需。
- SQL 已确认只读，结果已脱敏且能关联到真实代码/日志证据。
- “可用”状态有安全只读探测成功证据；仅发现配置时必须写“有证据但需配置”。
- 未把缺失记录、空值或连接失败自行解释为业务成功/失败。

## 失败处理

- 未授权或连接失败：保留脱敏错误与接入条件，不尝试绕过权限。
- 物理模型与运行库不一致：记录差异并停止猜字段，不执行修复 SQL。
- 查询结果不足以闭环：标记为当前推测，列出下一步最小只读取证。
