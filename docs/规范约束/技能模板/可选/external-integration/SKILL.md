# 接入第三方服务

## 触发条件

- 代码或需求存在真实第三方 API、SDK、Webhook/回调、签名验签、加解密或外部字段映射边界。
- 官方资料、现有集成或用户提供契约至少有一项可读取；版本、字段或环境不明时列入待补信息，不按常识推导。

## 执行步骤

1. 读取官方文档、官方 SDK/示例、项目已有同类集成和 `{{第三方集成文档}}`。
2. 固化接口、版本、环境、鉴权、请求头、字段、响应、错误、幂等和回调契约。
3. 优先复用项目现有 Client、Handler、配置、签名、序列化和错误映射能力。
4. 对官方字段逐项建立“官方字段 → 项目字段 → 来源/规则”映射，不凭名称猜语义。
5. 覆盖签名/验签、加解密、超时、失败响应、重复回调和乱序场景。
6. 敏感配置只引用配置键，不写入文档、日志或代码。
7. 更新第三方集成文档并记录官方资料版本和验证证据。

## 禁止事项

- 官方契约不明确时停止字段推断；禁止用“默认成功”、吞错、未授权重试或自创降级保持流程继续。
- 禁止写入密钥、token、证书私钥、生产账号或未脱敏请求响应；禁止另起平行 Client、签名、序列化或错误体系。

## 可执行验证

- 仅运行项目资源中确认存在的离线契约测试、签名验签、沙箱或集成测试命令；记录版本、环境、退出码、脱敏输入输出和范围。
- 对适用链路验证正常、超时、失败响应、签名错误、重复/乱序回调及幂等；无法验证的项进入待补信息。

## 完成 Gate

- 接口版本、鉴权、字段、错误、状态、幂等和回调契约都有官方或源码证据。
- 已复用项目既有 Client、配置、签名、序列化和错误体系，没有平行框架。
- 正常、超时、失败响应、签名错误、重复/乱序回调和敏感信息边界已覆盖。
- 文档只记录配置键与脱敏证据，不包含密钥、token、证书私钥或生产账号。

## 失败处理

- 官方资料不可访问或版本不明：停止字段推断，列出最少待确认项。
- 沙箱/测试环境不可用：保留离线契约测试结果，明确真实链路未验证。
- 第三方行为与文档冲突：保存请求/响应证据并标记差异，不擅自增加兜底。
