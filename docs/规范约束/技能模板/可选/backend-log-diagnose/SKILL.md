# 使用日志诊断后端问题

## 触发条件

- 已有后端问题和至少一个可核对的环境、服务、时间窗口、请求 ID 或业务号线索。
- 仅在日志入口已由代码、配置或安全只读探测证实时使用；配置键不是日志可用性的证明。

## 待补信息

初始化只生成待接入清单，不保存 token、密码、证书或生产日志。由开发者在获得授权后补充并验证：

{{日志能力补充表}}

Kubernetes 常用最小只读探测：

```bash
kubectl auth can-i get pods -n <namespace>
kubectl auth can-i get pods/log -n <namespace>
kubectl get pods -n <namespace> -l <label-selector>
kubectl logs -n <namespace> <pod> -c <container> --tail=1
```

Docker、本地文件、Loki/ELK/Grafana/云日志和 SSH/API/CLI 也必须以等价的最小只读命令验证。命令退出码、环境、服务、验证时间和脱敏结果摘要是验收证据。

## 执行步骤

1. 确认环境、时间窗口、服务、请求 ID/业务号和敏感信息边界。
2. 先用最窄条件检索，再围绕首个异常向前后扩展上下文。
3. 将日志时间线映射到入口、方法、数据状态和外部调用。
4. 必要时结合只读数据库证据；无授权不连接、不猜测。
5. 输出日志证据、代码证据、数据证据及结论可信度。

## 禁止事项

- 禁止泄露 token、密钥、完整证件号、银行卡号、用户隐私或生产敏感日志。
- 禁止越权读取、修改、删除日志/配置，禁止因无结果而无边界扩大扫描或臆想根因。

## 可执行验证

- 仅运行项目资源中已确认、且只读的日志命令；记录真实命令、环境、时间范围、退出码与脱敏结果摘要。
- 未完成最小只读探测时，能力状态只能是“有证据但需配置”或“未配置”。

## 完成 Gate

- [ ] 日志读取工具或入口真实存在，且当前 Agent 可执行。
- [ ] 环境、服务/Deployment、namespace、label selector、container 或日志文件路径已建立映射。
- [ ] 最小只读探测成功，真实命令退出码为 0，并记录验证时间与环境。
- [ ] 已验证读取权限边界，不具备写入、删除、配置变更或越权访问能力。
- [ ] 输出已脱敏，skill 和文档未保存凭证、个人信息或完整业务敏感数据。
- [ ] 已把接入方式、最小命令和验收证据写回本 skill；未全部满足时状态仍为“有证据但需配置”。

诊断任务本身还必须满足：已说明环境、服务、时间窗口和检索条件；每项结论能定位到日志片段与 `{{源码路径}}`；已区分“已确认根因”和“当前推测”。

## 失败处理

- 日志源未配置或无权限：保留真实错误与接入条件，继续完成代码可证明的部分。
- 检索无结果：核对环境、时区、实例、标识和采集范围，不扩大成无边界全量扫描。
- 多实例时间线冲突：按请求 ID/业务号重建顺序，无法确认时不得给确定根因。
