# 查找并复用项目技能

## 触发条件

- 用户问“有没有现成 skill”“该用哪个 skill”“是否需要新增 skill”，或任务开始前需要确认可复用工作流。
- 仅搜索当前项目已安装的 `.claude/skills` 与其软链接入口；外部 skill 只有在用户明确要求且环境允许时才考虑。

## 项目资源

| 资源 | 当前项目位置/状态 |
|---|---|
| Claude skills | `.claude/skills/` |
| Agent 软链接入口 | `.agents/skills/`（若项目已配置） |
| 规则索引 | `{{规则索引路径或未配置}}` |
| 技能索引/说明 | `{{skill 索引或未配置}}` |

## 执行步骤

1. 读取 `.claude/skills` 下每个 `SKILL.md` 的 frontmatter、触发条件和完成 Gate；确认 `.agents/skills` 是否只是同一来源的软链接。
2. 将用户任务拆为真实工作目标、技术层、风险和所需证据，逐项匹配已有 skill 的触发条件。
3. 输出“直接复用 / 组合复用 / 不适用”结论，并说明每项的路径、匹配依据、覆盖范围和缺口。
4. 只有现有能力无法覆盖且任务确有长期复用价值时，先使用 `skill-designer` 设计新增 skill；项目事实必须来自当前项目可读取的代码、配置或资料。

## 禁止事项

- 禁止只凭目录名判断技能能力，禁止把 `.agents/skills` 与 `.claude/skills` 的软链接重复计数。
- 禁止因为能找到通用 skill 就忽略项目规则、项目专属 skill 或已有实现。
- 禁止在没有项目证据或用户授权时安装外部 skill、复制其他项目 skill 或新建 skill。

## 可执行验证

- 记录实际读取的 `SKILL.md` 路径和匹配结论；若运行检索命令，只能使用项目中已确认存在的只读命令并记录退出码。
- 复用结论必须能说明已有 skill 覆盖了哪些步骤、哪些仍需项目事实或用户材料。

| 缺失信息 | 已确认事实 | 影响 | 后续证据 |
|---|---|---|---|
| {{skills 目录/任务范围/规则索引/外部安装授权}} | {{已发现 skill 路径}} | {{无法判断的复用范围}} | {{应读取的 SKILL.md、规则或用户确认}} |

## 完成 Gate

- 已区分实际可复用 skill、软链接重复项与不适用项。
- 没有创建、安装或修改任何 skill，除非用户后续明确授权。
- 新增需求已明确交由 `skill-designer`，或已写明不新增的事实依据。

## 失败处理

- `.claude/skills` 不可读或软链接异常：报告具体路径和系统错误，不把目录缺失解释为“没有可复用 skill”。
- 任务描述不足以匹配：列出已检索范围与最小待补信息，不擅自安装、复制或创建 skill。
