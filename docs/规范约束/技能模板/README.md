# 项目技能模板

本目录只保存跨项目可复用的 Agent 工作流模板。初始化时必须先识别项目类型、技术栈、已有规范与真实能力，再选择、填充并写入目标项目；禁止把本目录整体复制到目标项目。

## 选择规则

| 模板 | 生成条件 | 作用 |
|---|---|---|
| `公共/skill-designer` | 所有项目，原样安装 | 统一项目后续 skill 的模式选择、结构和质量标准 |
| `公共/find-skills` | 已安装项目 skills 或用户询问可复用能力 | 先查已有 project skills，避免重复创建或安装外部 skill |
| `公共/detail-design-writer` | 所有项目 | 从需求与现状形成可开发、可验证的详设 |
| `公共/developer` | 所有项目 | 按详设完成 TDD、实现、验证、文档同步与可选交付 |
| `公共/problem-diagnose` | 所有项目 | 从症状和证据定位根因，区分事实与推测 |
| `公共/code-review` | 所有项目 | 审查正确性、回归风险、复用与测试证据 |
| `公共/review-feedback-handler` | 有详设或代码评审流程 | 逐条核对、分类并处置评审意见 |
| `前端/frontend-self-test` | 检测到前端代码 | 覆盖组件、交互、请求状态与构建验证 |
| `后端/backend-self-test` | 检测到后端代码 | 覆盖单元、集成、异常、边界与原 Bug case |
| `可选/backend-log-diagnose` | 检测到后端 | 生成日志来源待接入清单；只有最小只读探测成功并通过 Gate 后标记可用 |
| `可选/database-read-diagnose` | 检测到数据库选型、依赖或连接配置 | 生成数据库只读能力待接入清单；配置键不等于可用，仅探测成功后标记可用 |
| `可选/ddl-review` | 检测到关系型数据库和迁移机制 | 审查表、字段、索引、迁移与兼容性 |
| `可选/external-integration` | 检测到第三方 API/SDK/回调 | 按官方契约审查请求、响应、安全与回调 |

## 初始化要求

1. 先原样安装 `skill-designer`；项目没有同名技能时不得省略。
2. 再复用项目已有同名或同职责 skill；存在时补充，不得重复创建。
3. 新建或改写项目 skill 时必须先使用 `skill-designer` 选择设计模式。
4. 稳定目录名、skill `name` 和 `SKILL.md` 文件名使用英文 kebab-case；正文标题、触发条件、资源、步骤、禁止事项、验证和待补信息使用中文。
5. 把模板中的项目事实替换为代码中可验证的真实内容；不能确认的内容只能写在“待补信息”，不得编造。
6. 删除不适用章节和未填占位符，目标项目中的 skill 必须可直接执行；没有确认的命令不写入“可执行验证”。
7. skill 只描述工作流和判断门槛；项目长期事实引用 `docs/`，强制约束引用 `rules/`。
8. 每个 skill 只保留 `SKILL.md` 及真正需要的脚本或引用，不生成额外说明文件。
9. 项目工厂只安装并填实 skills，不触发详设、开发、自测或 Git 交付；这些流程只在后续用户明确发起任务时执行。
10. 日志、数据库、SSH、云平台等外部能力遵循“安全探测成功才可用”；只发现配置键或部署线索时必须保留“待补信息”和完成 Gate。
