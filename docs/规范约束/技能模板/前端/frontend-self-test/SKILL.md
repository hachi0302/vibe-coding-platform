# 前端自测

## 触发条件

- 已发生前端页面、组件、路由、状态、请求交互或样式行为变更，且用户要求或交付需要自测。
- 浏览器、测试框架、视觉基线或真实接口不可用时如实记录，不用孤立 mock 页面替代项目运行验证。

## 执行步骤（Pipeline）

1. 读取前端规则、组件规范和详设验收项。
2. 先补能失败的测试，再实现修复或功能。
3. 使用项目已有测试框架和 mock 方式；优先测用户可见行为，不锁死内部实现。
4. 对关键页面做真实运行验证，记录视口、步骤和截图/结果；无浏览器能力时明确缺口。
5. 汇报测试文件、真实命令、通过数量、失败处置和遗留盲区。

## 可视化验收 Gate

- 使用当前项目实际启动方式打开目标页面，而不是孤立 HTML mock。
- 核对正常、加载、空态、失败、无权限、禁用和重复提交。
- 核对项目支持的主题、长文本、目标视口、键盘和可访问性。
- 截图只能证明展示，交互结果还必须有操作步骤、请求/状态与自动化证据。

## 禁止事项

- 禁止更新快照、删除断言、扩大 mock 范围或跳过失败状态来求绿。
- 禁止臆造页面路径、组件规范、浏览器能力、接口响应、测试命令或视觉基线。
- 禁止把截图、构建成功或单个交互成功当作全部状态已验证。

## 可执行验证

- 仅运行项目资源中确认存在的 lint、类型检查、测试、构建和启动命令；记录命令、cwd、退出码、通过数、视口与验证步骤。
- 对适用变化核对正常、加载、空态、失败、无权限、禁用和重复提交；不可执行的状态进入待补信息。

## 失败处理

测试失败不得更新快照、删除断言或扩大 mock 范围求绿；必须先判断是 production 缺陷、测试错误还是环境阻断，并记录处置。

## 完成 Gate

测试路径、case 矩阵、真实命令、通过数量、可视化结果、失败处置与测试文件去留全部可复查后才算完成。

编译成功不等于交互正确；快照通过不等于关键状态已覆盖。
