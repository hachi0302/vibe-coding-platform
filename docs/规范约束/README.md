# 项目初始化规范源

本目录是 Vibe Coding Platform 初始化项目时使用的唯一规范源，只包含三类内容：

| 目录 | 职责 | 是否直接复制到目标项目 |
|---|---|---:|
| `文档模板/` | 约束长期文档与迭代文档的目录、章节和证据格式 | 只复制适用的三个规范模板；长期文档由 Agent 读取项目后填实 |
| `规则模板/` | 约束 Agent 修改代码时必须遵守的触发规则 | 不直接照抄；根据项目技术栈、框架和现有实现生成项目规则 |
| `技能模板/` | 定义可执行的开发、诊断、评审和自测 Pipeline | 不整库复制；只生成当前项目适用且职责不重复的 skills |

## 初始化红线

1. 先完整读取项目，再生成项目事实；模板不能冒充项目文档。
2. 前端、后端、全栈按真实代码层生成，不创建不适用的空壳。
3. 不生成运维、部署、监控告警、服务器维护或 CI 运维资料。
4. 已有文档、规则、skills 默认保留；没有平台标识的文件视为用户资产。
5. `.claude/` 与 `CLAUDE.md` 是唯一维护源，`.agents/` 与 `AGENTS.md` 使用软链接。
6. 不确定的事实写明证据缺口，禁止自作主张添加默认值、静默兜底或伪成功。
