# 项目规则模板索引

规则用于约束“怎么修改当前项目”，不是项目说明书，也不是工作流 skill。初始化时必须把占位符替换为项目真实目录、类、函数、命令和调用证据；没有对应代码层或能力时不生成该规则。

## 触发条件 → 必读规则

| 任务/代码 pattern | 必读规则 | 生成条件 |
|---|---|---|
| 任意代码修改 | `公共/开发基线.md`、`公共/复用与影响面.md`、`公共/事实与兜底边界.md` | 固定 |
| 新需求/Bug/代码优化 | `公共/开发流程与文档同步.md`、`公共/自测与交付.md` | 固定 |
| Git 分支/worktree/提交/推送 | `公共/Git协作与历史保护.md` | 固定 |
| 前端页面、路由、组件、状态、请求 | `前端/前端工程规则.md`、`前端/前端验证规则.md` | 检测到前端 |
| 后端 API、Service、错误、权限 | `后端/API与业务实现规则.md` | 检测到后端 |
| 实体、表、查询、迁移 | `后端/持久化与迁移规则.md` | 检测到数据库 |
| 消息、任务、外部 SDK/API、回调 | `后端/异步与第三方规则.md` | 检测到对应能力 |

规则正文必须给出项目证据，不能只写“遵守最佳实践”。
