# 异步与第三方规则

> 仅检测到消息、任务、外部 SDK/API 或回调时生成，并删除不适用章节。

## 触发条件

修改消息生产或消费、异步任务、外部服务调用、软件包集成、签名验签、重试、回调、通知或状态映射时必须阅读。

## 先读与复用优先

1. 先阅读当前项目已有的生产入口、消费入口、注册配置、请求封装、回调处理、错误处理和真实成功或失败链路。
2. 外部字段、签名、验签、状态映射和回调响应优先以官方资料、软件包源码、真实报文或项目已有成功链路为准。
3. 新增集成前先查是否已有同一服务、同一协议或同一业务动作的封装；可扩展时不得新建平行客户端或回调入口。

## 强制规则

1. 按现有生产、消费、重试、幂等和错误可见性模式实现，不另建平行框架。
2. 外部字段、签名、验签、状态映射和回调响应必须来自官方文档、SDK、真实报文或已有成功链路。
3. 禁止添加官方未定义的字段优先级、默认值、静默重试、模拟成功或多来源兜底。
4. 回调必须明确认证/验签、幂等键、重复通知、状态保护、失败响应和重试影响。
5. 新的可复用集成约束同步到对应专项 skill 与第三方集成长期文档。

## 禁止事项

- 禁止添加官方或现有契约未定义的字段优先级、默认值、静默重试、模拟成功、隐式降级或多来源兜底。
- 禁止跳过认证、验签、幂等、重复通知、状态保护或失败可见性检查。
- 禁止只修改调用端或回调端一侧，遗漏配置、注册、生产、消费、状态更新、测试或长期资料。
- 禁止把不可访问的外部系统、未知报文或未确认配置描述为已验证可用。

## 验证要求

- 验证成功、超时、异常、重复消息或重复回调、未知状态和认证失败等真实分支。
- 核对生产与消费、请求与回调、签名与验签、重试与幂等在同一真实契约下闭环；无法访问外部系统时记录替代验证和剩余风险。
