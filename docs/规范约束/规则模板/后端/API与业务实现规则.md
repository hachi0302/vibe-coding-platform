## 强制规则

1. API 入口只承担项目现有框架允许的鉴权、校验、转换和调用，不把业务逻辑塞进入口层。
2. 业务实现复用现有错误码、异常基类、日志格式、转换器和上下文；禁止裸抛不受控异常或返回伪成功。
3. 修改契约时同步所有调用方、类型、测试与 API 文档；兼容策略必须明确到字段和版本。
4. 状态、金额、时间、精度、并发和幂等按项目已有模型处理，不发明新的表达方式。
5. 日志记录定位所需关键参数并按项目规则脱敏，禁止输出密钥、令牌和完整敏感字段。
