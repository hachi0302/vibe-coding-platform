# 持久化与迁移规则

## 触发条件

修改实体、数据访问、查询、表、字段、索引、迁移、事务、数据初始化或数据库配置时必须阅读。

## 先读与复用优先

1. 先阅读当前表或集合的真实定义、迁移记录、实体映射、数据访问层、调用方和同类查询或写入实现。
2. 优先复用已有实体、仓储、查询条件、分页、转换、迁移目录和事务边界；新增前确认不存在同语义表、字段或访问入口。
3. 表、字段、索引、默认值、精度、时区和空值语义只以当前项目的 schema、迁移和实体证据为准。

## 强制规则

1. 新增查询或写入前检索现有 Repository/DAO/Mapper 和实体映射，复用已有字段、转换和分页方式。
2. 表、字段、索引、默认值、精度、时区和空值语义以 schema/迁移/实体证据为准。
3. 已执行的历史迁移不可修改；新迁移按项目版本命名并提供前向与回滚影响说明。
4. 事务边界与异常回滚遵守项目现有框架；跨边界一致性不能靠吞错或假成功实现。
5. 变更前全局检索表/字段引用并同步简化物理模型、API、测试和相关长期文档。

## 禁止事项

- 禁止修改已执行的历史迁移、重写既有 schema，或仅根据实体推测真实数据库结构。
- 禁止新建重复表、重复字段、平行仓储或绕过项目既有数据访问和迁移机制。
- 禁止猜测默认值、索引、精度、字符集、时区、空值、事务或回滚语义。
- 禁止以吞异常、部分写入或假成功代替跨边界一致性处理。

## 验证要求

- 检查迁移、实体映射、数据访问、调用方和长期物理模型的字段及约束口径是否一致。
- 对新增或修改的查询、写入、索引和迁移运行项目真实的定向验证；无法连接数据库时明确记录未验证范围和风险。
