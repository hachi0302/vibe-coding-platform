# Git 协作与历史保护

1. 开始任务先查看当前分支、工作区状态和上游；不覆盖、不还原、不提交用户已有改动。
2. 多任务并行时可使用 worktree；是否使用由用户选择，不能因小任务强制创建。
3. 未经用户明确授权，不执行 commit、push、rebase、merge、cherry-pick 或创建 PR/MR。
4. 用户选择提交时，只暂存本任务文件；commit message 遵守项目现有规范与历史提交风格。
5. push 被拒绝时先 fetch 并展示远端差异；不得使用强推。发生冲突时解释每个冲突的两侧语义，保护双方有效改动。
6. 禁止改写共享分支历史；禁止删除未知分支、worktree、stash 或未跟踪文件。
