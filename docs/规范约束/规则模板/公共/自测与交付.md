# 自测与交付

## 触发条件

完成代码、配置、数据结构、接口、前端交互或缺陷修复，准备交付、提交或发布前必须阅读。

## 先读与复用优先

1. 先查现有测试、测试数据、构建脚本、质量检查和交付记录，复用项目真实的验证入口。
2. 修复缺陷时优先保留或新增能够复现原问题的测试；新增功能时复用同类功能的测试组织与断言方式。

## 强制覆盖

每次业务代码变更至少检查：正常、边界、异常、原 Bug（修复时）和直接相关历史回归。测试必须运行真实项目代码，不能只验证 mock 调用次数或自造等价逻辑。

## 验证要求

1. 写失败测试并确认失败原因正确。
2. 完成最小实现，运行定向测试。
3. 运行模块测试、静态检查、编译/构建及项目要求的必要全量测试。
4. 记录测试路径、完整命令、退出码、关键结果和测试文件处置。
5. 无法执行时说明阻断、替代验证与剩余风险；禁止写“应该通过”。

## 禁止事项

- 禁止伪造命令、退出码、测试结果、截图、日志或“已验证”结论。
- 禁止仅验证模拟调用、复制实现或静态文本，而不验证真实项目代码的关键行为。
- 禁止以未执行的全量检查掩盖定向测试、构建或兼容性风险。
