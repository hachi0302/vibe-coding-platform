# 开发基线

## 触发条件

修改任意代码、配置、测试、构建脚本或项目长期文档前必须阅读。本规则只写当前项目已确认的语言、框架、目录、分层和命名约定。

## 先读与复用优先

1. 修改前依次阅读目标文件、调用入口、直接调用方、同目录同职责实现和命中的专项规则。
2. 新增类型、常量、组件、工具、配置或脚本前，先全局检索同名、同职责和相似实现；至少阅读一个真实调用示例后再决定复用或扩展。
3. 现有能力可扩展时优先扩展，不创建平行入口、重复包装或只为当前任务服务的公共层。

## 强制规则

1. 只改当前任务直接相关内容；不顺手重构、改格式、批量重命名或清理用户未提交改动。
2. 项目技术栈、目录、公共框架和真实命令以 `{{项目证据路径}}` 为准，禁止用通用示例覆盖当前项目做法。
3. 新增代码遵循当前项目已有的命名、分层、依赖方向、错误表达和测试组织方式；没有证据时先记录待补信息。
4. 注释解释业务意图、约束或不可见原因，不写翻译代码、外部项目名、任务编号和决策过程。

## 禁止事项

- 禁止根据个人偏好替换项目既有框架、目录结构、封装层或代码风格。
- 禁止把别的项目的类名、模块名、接口路径、命令或业务约束复制为当前项目事实。
- 禁止为了“统一”修改无关文件、覆盖未提交改动或制造无法说明的公共抽象。

## 验证要求

- 检查新增或修改内容与同类真实实现的目录、命名、调用方向和错误处理是否一致。
- 运行当前项目已确认且与改动匹配的验证命令；未配置命令时只记录验证目标和阻断原因，不伪造命令。
