# 开发基线

## 适用范围

所有代码、配置、测试和项目文档修改。

## 强制规则

1. 改文件前先读目标文件、上游入口、下游调用方、同目录现有实现和命中的规则。
2. 只改当前任务直接相关内容；不顺手重构、改格式、批量重命名或清理用户未提交改动。
3. 项目技术栈、目录、公共框架和真实命令以 `{{项目证据路径}}` 为准，禁止用通用示例覆盖当前项目做法。
4. 注释解释业务意图、约束或不可见原因，不写翻译代码、外部项目名、任务编号和决策过程。
5. 新增代码必须符合当前项目已有的命名、分层、依赖方向、错误表达和测试组织方式。
