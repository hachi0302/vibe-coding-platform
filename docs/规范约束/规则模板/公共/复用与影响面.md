# 复用与影响面

## 触发条件

新增或修改公共能力、共享模型、枚举、错误表达、配置、序列化、请求封装、数据结构或跨模块调用时必须阅读。

## 先读与复用优先

1. 全局检索同名、同职责、同数据结构和同业务动作，不只看用户点名文件。
2. 优先复用当前项目已识别的公共能力目录，并读取至少一个真实调用示例。
3. 新增公共能力前说明现有能力为什么不能扩展；无法说明则不新增。

## 修改时

- 改公共组件、共享模型、枚举、错误码、缓存 Key、序列化或请求封装前，列出所有引用方。
- 改 API/事件/数据结构时，同步检查生产方、消费方、类型定义、测试和长期文档。
- 对历史行为给出“不变清单”，测试至少覆盖一个直接相关回归场景。

## 禁止事项

- 禁止仅因名称不同就复制已有能力、类型、常量或请求封装。
- 禁止变更共享契约后只修改当前调用点，遗漏生产方、消费方、类型、配置、测试或文档。
- 禁止以空值、默认值、重复实现或静默兼容掩盖影响范围不清的问题。

## 验证要求

- 在修改前后分别检查真实引用方，确认新增或扩展能力没有制造重复入口。
- 对受影响调用链执行定向验证；涉及公开契约时核对请求、响应、错误和兼容行为。
