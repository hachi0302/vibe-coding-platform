# 前端工程规则

## 触发条件

修改页面、路由、布局、组件、状态、请求封装、类型、样式、国际化、权限、前端配置或前端构建脚本时必须阅读。

## 先读与复用优先

1. 先阅读目标页面或组件、路由入口、布局容器、状态入口、请求封装、类型定义和同类真实页面。
2. 新增页面、组件、组合逻辑、状态、类型、常量或样式前，全局检索同职责实现；优先复用已有公共能力并读取真实调用示例。
3. 只有现有能力无法扩展且证据充分时，才新增公共能力；新增前说明适用范围和与现有能力的边界。

## 强制规则

1. 页面按现有路由、布局、状态和请求层接入，不在组件内重复创建客户端或全局能力。
2. 优先复用已有组件、hooks/composables、类型、常量和样式 token；新增公共组件前全局检索。
3. 服务端数据、共享状态、URL 状态和组件本地状态按当前项目边界存放，不为方便随意全局化。
4. API 类型、枚举、权限和错误展示与真实契约一致；未知值不默认为成功或第一个选项。
5. 保持现有视觉语言、交互反馈、键盘/可访问性和响应式规则；不因单一截图破坏已有组件体系。

## 禁止事项

- 禁止在页面或组件内部重复创建全局客户端、状态容器、请求拦截或主题能力。
- 禁止根据接口名称、页面截图或常见框架习惯猜测字段、枚举、权限、路由或交互。
- 禁止把服务端数据、共享状态和组件局部状态混放，或以默认成功、默认首项、空对象掩盖未知数据。
- 禁止为了当前页面绕开已有布局、组件、样式令牌、权限或错误处理体系。

## 验证要求

- 验证页面或组件在真实路由、权限、加载、空态、失败态和关键交互下符合项目已有行为。
- 涉及请求时核对真实参数、响应映射、枚举展示和错误处理；涉及共享能力时检查真实引用方。
